# 内容诊断（改造版）

> 参考来源：dbskill 的 `dbs-content` 与内容知识包，已压缩为 OpenClaw 适用版。

## 一句话目标

先判断“这内容值不值得这样写”，再动笔。

## 先看 4 件事

1. **这内容讲什么**
2. **写给谁**
3. **平台/文体是什么**
4. **目标是什么**（点击 / 留存 / 转化 / 观点表达）

## 6 个诊断维度

### 1. 文字洁癖
看有没有这些问题：
- 空话太多
- AI 腔太均匀
- 概念大于事实
- 句子都像“正确答案”

### 2. 标题 / 封面承载力
看标题和第一眼信息能不能让人判断“值不值得继续看”。

### 3. 表达效率
看有没有：
- 30 句话只讲 1 个意思
- 99% 在包装，1% 是内容
- 用抽象词偷懒

### 4. 认知落差
看这篇内容有没有“我不看别家也得看你”的理由：
- 新信息
- 新角度
- 更强表达
- 更高密度

### 5. 平台匹配
- 小红书：图文/标题/封面判断权重高
- 抖音：口播友好、开头强度、情绪推进更重要
- 公众号：逻辑链和节奏更重要
- X / 推文：观点锐度、可引用句、短促张力更重要

### 6. 情绪势能 / 作者存在感
看有没有这些问题：
- 文字正确，但没情绪
- 观点存在，但下手太轻
- 全文都在解释，没有一句能留下来
- 改完会更稳，但更不像本人

## 常见结论模板

### 可做
- 方向对
- 素材够
- 主要问题是表达方式

### 不能直接做
- 选题太空
- 受众不清
- 平台错配
- 没有任何具体素材，只有态度

## 推荐输出格式

- 这篇内容在讲什么
- 最大问题是什么
- 该保留的情绪 / 观点 / 口气是什么
- 最该先改哪一刀
- 如果重写，优先改：标题 / 开头 / 结构 / 语气 / 例子

## 诊断后的动作建议

- 标题弱 → 去 `title-patterns.md`
- 开头弱 → 去 `hook-patterns.md`
- AI 味重 → 去 `ai-fingerprint.md`
- 全文都虚 → 先补事实、案例、数字、具体场景，再重写
